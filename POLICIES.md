# Public Policies

FacturApp keeps the public story simple and the private implementation private.

## Privacy and data

- User-facing documentation is public.
- Source code and internal deployment details stay private.
- Secrets, keys, and provider credentials are never published.
- Customer data is handled through the product, not documented as public content.

## Payments

- Payments use Stripe.
- FacturApp does not hold client funds.
- We do not present the product as a bank or money transmitter.

## Product boundaries

- We describe the product as invoicing and payment workflow software.
- We do not position it as a full accounting replacement.
- We avoid promising features the product does not actually expose.

## GitHub plan policy

- Public presentation should stay on GitHub Free.
- We use public repositories and profile README surfaces only.
- No paid GitHub feature is required to present the brand well.

## Implementation boundary

Public docs explain what users can do and what policies apply.
They do not disclose source code, private APIs, or operational secrets.
